# Implementation Plan

## table-of-contents

---

- [x] 1. 依存関係の追加と型定義のセットアップ

- [x] 1.1 (P) github-slugger を直接依存関係として追加する
  - `github-slugger` を `package.json` の `dependencies` に追加してインストールする
  - 現在トランジティブ依存として存在しているため、インストール自体の追加コストはほぼゼロ
  - _Requirements: 1.2_

- [x] 1.2 (P) 目次アイテムの型を定義する
  - 見出しを表す型（`id: string`・`text: string`・`level: 2 | 3`）を新規ファイルに定義する
  - この型はサーバー側のユーティリティとクライアントコンポーネントの両方が参照する共有インターフェース
  - _Requirements: 1.1, 1.4_

---

- [x] 2. (P) Markdown 見出し抽出ユーティリティを実装する
  - `src/lib/` に新規モジュールとして実装する（`extractTocItems` 関数）
  - Markdown 本文の各行を解析し、`##` で始まる h2 行と `###` で始まる h3 行を抽出する
  - `github-slugger` の `GithubSlugger` クラスを使って各見出しの ID を生成し、`rehype-slug` が付与する ID と一致させる
  - ID が空文字になる見出し（英数字を含まない日本語のみ見出し等）を除外する
  - `text`・`id`・`level` を持つ目次アイテムの配列を返す
  - 入力が空文字や見出しなしの場合は空配列を返す
  - _Requirements: 1.1, 1.2, 1.4_
  - ※ タスク 3.1 と並列実行可能（別ファイル・相互依存なし）

---

- [x] 3. TableOfContents コンポーネントを実装する

- [x] 3.1 (P) 目次リストとジャンプリンクを実装する
  - `src/components/post/` に `'use client'` クライアントコンポーネントとして新規作成する
  - `TocItem[]` を受け取り、「目次」ラベルとアンカーリンクのリストを表示する
  - h3 アイテムは h2 より字下げしてインデントで階層を表現する
  - 各リンクの `href` に `#{id}` 形式のアンカーを設定する
  - スムーズスクロールは CSS `scroll-behavior: smooth` で対応し（グローバル CSS に追加）、JS での制御は行わない
  - ※ タスク 2 と並列実行可能（別ファイル・相互依存なし）
  - _Requirements: 2.1, 2.2, 2.4_

- [x] 3.2 スクロール連動ハイライトを実装する
  - `useEffect` 内で `IntersectionObserver` を生成し、記事本文内の h2・h3 要素を監視する
  - `rootMargin: '-80px 0px -70% 0px'` でビューポート上部付近の帯域を検知エリアに設定する
  - 見出しが検知帯域に入ったときに `activeId` state を更新し、対応する目次アイテムをハイライト表示する
  - ページロード時は最初の見出しを初期アクティブ状態にする
  - `useEffect` の return で `observer.disconnect()` を呼んでクリーンアップする
  - `IntersectionObserver` が未サポートの環境では監視をスキップし、ハイライトなしで表示する
  - _Requirements: 3.1, 3.2, 3.3_

- [x] 3.3 レスポンシブ対応とモバイル折りたたみを実装する
  - `isOpen` state（初期値 `false`）でモバイルでの展開・折りたたみ状態を管理する
  - Tailwind の `lg:` プレフィックスを使い、デスクトップとモバイルのスタイルを切り替える
  - モバイル（`lg` 未満）ではデフォルト折りたたみ状態でトグルボタンを表示し、クリックで展開・折りたたみできるようにする
  - デスクトップ（`lg` 以上）ではモバイル専用のトグルボタンと折りたたみ UI を非表示にする
  - _Requirements: 4.2, 4.3, 4.4_

- [x] 3.4 アクセシビリティ属性を追加する
  - 目次全体を `<nav aria-label="目次">` 要素でラップする
  - モバイルのトグルボタンに `aria-expanded` 属性を付与し、`isOpen` state と連動させる
  - アクティブな目次アイテムに `aria-current="true"` を付与し、非アクティブ時は除去する
  - すべてのリンクに `<a>` タグを使用し、キーボード（Tab・Enter）でフォーカス・クリック操作が可能なことを確認する
  - _Requirements: 5.1, 5.2, 5.3, 5.4_

---

- [x] 4. 記事詳細ページへの統合

- [x] 4.1 extractTocItems を呼び出して TOC データを生成し条件付きレンダリングを実装する
  - `src/app/posts/[slug]/page.tsx` で `extractTocItems(post.content)` を呼び出して目次データを生成する
  - 目次アイテム数が 2 未満の場合は TOC コンポーネントを一切レンダリングしないように条件分岐を設ける
  - `extractTocItems` の呼び出しを `try/catch` で囲み、例外発生時は空配列にフォールバックしてページ表示を継続する
  - _Requirements: 1.3_

- [x] 4.2 デスクトップ 2 カラムレイアウトに変更して TOC サイドバーを配置する
  - 記事詳細ページのアウターコンテナを `lg:flex lg:items-start lg:gap-10` の 2 カラムレイアウトに変更する
  - 記事本文コンテナを `flex-1 min-w-0` に変更して TOC サイドバーとの並列配置に対応させる
  - モバイル用の TOC（`lg:hidden`）を記事本文の先頭に配置する
  - デスクトップ用の TOC サイドバー（`hidden lg:block`）を `sticky top-24` でスティッキー配置する
  - 目次アイテムが 2 件以上ある場合のみ、両方の TOC 配置をレンダリングする
  - _Requirements: 2.3, 4.1_
