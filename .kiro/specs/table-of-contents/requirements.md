# Requirements Document

## Project Description (Input)
目次（TOC）自動生成機能

## Introduction

本機能は、記事詳細ページに目次（Table of Contents）を自動生成・表示する機能である。
長い技術記事において、読者が全体構成を素早く把握し、読みたいセクションへ直接ジャンプできるようにする。
既存の `rehype-slug` によって見出しに `id` が付与されているため、アンカーリンクの基盤は整っている。
目次はスクロール位置に連動してアクティブ状態を示し、レスポンシブ対応・アクセシビリティにも配慮する。

---

## Requirements

### Requirement 1: 見出しの抽出

**Objective:** ブログ読者として、記事の見出し構造から目次を自動生成してほしい。そうすれば、長い記事の全体構成を一目で把握できる。

#### Acceptance Criteria

1. When 記事詳細ページが表示されるとき、the ブログシステムは Markdown 本文から h2・h3 レベルの見出しを抽出しなければならない。
2. The ブログシステムは各見出しのテキストと対応するアンカー ID を取得しなければならない。
3. If 記事に h2・h3 の見出しが 2 つ未満の場合、the ブログシステムは目次コンポーネントを表示してはならない。
4. The ブログシステムは h2 と h3 の階層構造を保持した見出しデータ（テキスト・ID・レベル）を生成しなければならない。

---

### Requirement 2: 目次の表示と見出しへのジャンプ

**Objective:** ブログ読者として、目次リンクをクリックしてページ内の該当セクションへ移動したい。そうすれば、読みたい部分に素早くアクセスできる。

#### Acceptance Criteria

1. When 目次が表示されるとき、the ブログシステムは「目次」のラベルと h2・h3 の階層に対応したインデント付きリンクリストを表示しなければならない。
2. When 読者が目次のリンクをクリックしたとき、the ブログシステムは対応する見出しへスムーズスクロールしなければならない。
3. The ブログシステムは目次を記事本文の前（本文上部）または横（サイドバー）に配置しなければならない。
4. The ブログシステムは目次の各リンクに、対応する見出しへのアンカー URL（`#slug`）を付与しなければならない。

---

### Requirement 3: スクロール連動ハイライト

**Objective:** ブログ読者として、スクロール中に現在読んでいるセクションが目次上でわかるようにしたい。そうすれば、長い記事でも自分の位置を見失わずに読み進められる。

#### Acceptance Criteria

1. While 読者がページをスクロールしているとき、the ブログシステムは現在ビューポートに入っているセクションに対応する目次アイテムをアクティブ状態でハイライト表示しなければならない。
2. When 記事詳細ページが最初に表示されるとき、the ブログシステムは最初の見出しを初期アクティブ状態にしなければならない。
3. When アクティブな見出しが切り替わったとき、the ブログシステムは旧アクティブ状態を解除し、新たなアクティブアイテムにスタイルを適用しなければならない。

---

### Requirement 4: レスポンシブ対応

**Objective:** ブログ読者として、スマートフォン・タブレット・PC のどのデバイスでも目次を適切に利用したい。そうすれば、閲覧環境に関わらず快適に記事を読める。

#### Acceptance Criteria

1. While デスクトップ幅（1024px 以上）のとき、the ブログシステムは目次を記事本文と並列にスティッキー表示しなければならない。
2. While モバイル幅（1023px 以下）のとき、the ブログシステムは目次を記事本文の上部に折りたたみ可能な形式で表示しなければならない。
3. When 読者が折りたたまれた目次のトグルボタンをクリックしたとき、the ブログシステムは目次を展開または折りたたまなければならない。
4. The ブログシステムはモバイル表示時にデフォルトで目次を折りたたみ状態にしなければならない。

---

### Requirement 5: アクセシビリティ

**Objective:** すべての読者として、支援技術（スクリーンリーダー・キーボード操作）を使用していても目次を利用したい。そうすれば、障害の有無によらず等しく記事を利用できる。

#### Acceptance Criteria

1. The ブログシステムは目次を `<nav>` 要素でマークアップし、`aria-label="目次"` を付与しなければならない。
2. The ブログシステムは目次内の全リンクをキーボード（Tab・Enter）で操作できるようにしなければならない。
3. When 折りたたみ目次のトグルボタンが表示されているとき、the ブログシステムは `aria-expanded` 属性で開閉状態を表現しなければならない。
4. The ブログシステムはアクティブな目次アイテムに `aria-current="true"` を付与しなければならない。
